Read Me for the Amazon Mechanical Turk SDK for Java v1.2.2

The Amazon Mechanical Turk SDK for Java is a set of libraries and tools designed to make it easier for you to build solutions leveraging Amazon Mechanical Turk. The goals of the SDK are:

  • To abstract you from the "muck" of using web services
  • To simplify using the various Amazon Mechanical Turk APIs
  • To allow you to focus more on solving the business problem and less on managing technical details

Version Information

This distributable contains the Amazon Mechanical Turk SDK for Java version 1.2.2.

The SDK is compatible with Amazon Mechanical Turk web service version 2007-06-21.

The Amazon Mechanical Turk team would appreciate any comments and feedback on the SDK.

Please submit issues on the SourceForge project.

SDK Contents

The SDK distributable contains the following content:
  • Overview of the Amazon Mechanical Turk SDK for Java to help you get started with the SDK.
  • Pre-compiled binaries ready to be used for development.
  • Set of sample applications demonstrating various aspects of the SDK and Amazon Mechanical Turk.
  • Javadoc style documentation for SDK
  • Source code for SDK and Apache Ant build script to rebuild the SDK.

Revision History

Version 1.2.2 - April 24, 2008

  • Bug Fix:1936985 - no results if the HIT has more than 10 assignments.
  • Bug Fix:1939778 - The number of retrieved results displayed is off by one
  • Bug Fix:1939779 - totalAssignmentsCompleted is double counted
  • Feature: 1899195 - Add a link to manageHIT page in results file

Version 1.2.1 - February 6th, 2008

  • New Feature: 1836115 - Add some additional line feeds when outputting the URL.
  • New Feature: 1836118 - GetResults should output rows with no assignments.
  • New Feature: 1836126 - always override .success, .results files.
  • New Feature: 1838902 - Add exponential back off to handle throttling.
  • New Feature: 1838914 - Change the default delimiter to tab.
  • Fixed: 1836109 - The generated .success file is a Unix file even in Windows.
  • Fixed: 1836111 - GetResults has missing columns when value is NULL.
  • Fixed: 1836114 - GetResults: Retrieved HIT number is misleading.

    • NOTE: The default delimiter is now tab. We reverted the change in the last version which made the default delimiter to be comma.
    NOTE: We migrated to using JaxMe 2 instead of JAXB. As a result, the auto-generated class names in aws-mturk-dataschema.jar are slightly different than the previous version. If your application directly depends on the classes in aws-mturk-dataschema.jar, you'll need to update your code due to these changes.
    NOTE: We're no longer providing junit as a part of the distributable. This means that you will have to download it for tests. This version of the SDK was built using junit 4.4. See the Overview for more information on using junit.

Version 1.2.0 - October 12th, 2007

  • New Feature: 1796739 - Added multi-threading support to batch functions in order to significantly speed up operations such as creating, updating and retrieving results of HITs.
  • New Feature: 1796673 - Added proper CSV support so that the CSV output file conforms to CSV file format standards by leveraging the opencsv library.
    NOTE: The default delimiter is now a comma and no longer defaults to a tab.

Version 1.1.1 - September 13th, 2007

  • New Feature: 1791360 - Update code to include new QuestionForm functionality.
  • Fixed: 1788853 - AssignQualification does not handle sendNotification flag.

Version 1.1.0 - August 20th, 2007

  • New Feature: Updated to the 2007-06-21 WSDL version.
  • New Feature: Support for ChangeHITTypeOfHIT, including updateHIT helper methods.
  • New Feature: Added GetAllQualificationRequests method to RequesterService.
  • New Feature: Removed singleton ClientConfig instance in RequesterService. You can now have multiple instances of RequesterService with different Configs in the same jvm instance.
  • New Feature: Added FilterChain functionality. Developers can write their own Filters which extend com.amazonaws.mturk.filter.Filter to handle requests, responses and errors from service calls.
  • New Feature: Added RetryFilter to retry on throttling errors. The Developer Sandbox will return throttling errors when the rate of requests exceed the limits.
  • New Feature: SDK will default to look for authentication information in $HOME/.aws/auth
  • Improvement: Included automated tests for the SDK.
  • Improvement: Deprecating config-less instantiation and refactoring Input/Output handlers.
  • Fixed: Adding UTF-8 support for writing output.

Version 1.0.1 - June 15th, 2007

  • New Feature: When retrieving results, the data is streamed directly to the Results file instead of being cached in memory and written at once. Avoids out of memory errors when retrieving large data sets.
  • New Feature: The generated Results file now orders the question/answer pairs based on the order they appear in the QuestionForm instead of being ordered alphabetically.
  • New Feature: Progress indicator when results are being retrieved from the system.
  • Fixed: Batch operations halt when the first error is encountered instead of skipping and continuing.
  • Fixed: Removed some extraneous files from the distribution.

Version 1.0.0 - June 4th, 2007

  • New Feature: User specified logging has been added to aid debugging and reporting issues to Amazon Mechanical Turk.
  • New Feature: SDK for Java supports the Amazon Mechanical Turk Developer Sandbox http://sandbox.mturk.com.
  • New Feature: The location of the mturk.properties file can be specified to the SDK classes.
  • Fixed: SetWorkerAcceptLimit operation is not compatible with Mechanical Turk.
  • Fixed: Retrieving assignments for a HIT over a certain number causes the code to lockup.
  • Fixed: Time format in Results file should be standard 0-23 hr format instead of 1-24 format.
  • Fixed: QuestionForm validation fails when using ExternalQuestion functionality
  • Fixed: QuestionForm validation does not validate the XHTML contained in the FormattedContent blocks of the QuestionForm. It will currently ignore content inside FormattedContent blocks.
  • Fixed: The ordering of the name/value pairs of the HIT results output file is inconsistent.
  • Fixed: When writing the .success and .failure files, it overwrites the file instead of appending.
  • Fixed: The .failure file is output if there are no failures. It should only exist if there are errors.

Version 0.9.0 Beta - March 27th, 2007
